# Инструкция по скриптам Roblox

Полный свод правил по созданию Roblox-скриптов. Читать и соблюдать при любой работе со скриптами.

## Код и скрипты
- Все скрипты Roblox делать строго тут: `D:\Нужное\Скрипты роблокс\Делаем скрипты тут\Скрипты`.
- Имя файла скрипта — **строго название самой игры** (без лишних приписок вроде «script», «hack», версий и т.п.).
- Имя файла — **строго на английском**, даже если игра называется на русском (использовать официальное англ. название игры; если его нет — перевести/транслитерировать). Никакой кириллицы в именах файлов.
- **Исключение — универсальные скрипты:** если прошу сделать универсальный скрипт (не под конкретную игру), имя файла — **не название игры**, а описание его функции/функций (на английском, без кириллицы). Напр. `Auto Farm.lua`, `ESP.lua`, `Infinite Yield.lua`.
- В коде и скриптах **не оставлять пометок/помарок**: никаких заметок-комментариев о правках («// fixed», «// добавил», «// было/стало»), TODO/FIXME-маркеров, подписей AI и прочих следов. Код чистый, как будто так и было написано.
- Скрипты Roblox делать **сразу с красивым GUI** (а не голый код) и **сразу с функцией unload** (выгрузка: закрыть/удалить GUI, отключить все соединения/лупы, снять хуки — чтобы скрипт можно было полностью убрать).
- **Сохранять состояние между перезапусками.** Все включённые функции и их настройки (что включено, значения — спам/сек, выбранные опции и т.п.) писать в конфиг (`writefile`/`readfile`, JSON) и **авто-восстанавливать при следующем запуске**. После перезапуска Roblox / перезахода / повторного запуска скрипта ранее включённые функции снова включаются сами, ничего перенастраивать не надо.
- **Не делать блюр/размытие фона за GUI** (никакого `DepthOfField`, `BlurEffect` и прочих backdrop-блюров как фон под окном GUI). Фон за GUI остаётся чистым — игра видна как есть.
- **Сначала проверять, нужен ли спам вообще.** Если у того, что спамится, есть лимит/кулдаун/ограничение, из-за которого спам бессмыслен (лишние вызовы просто отбрасываются сервером) — не спамить, а делать **умную покупку/действие по таймингу** (вызывать ровно в момент готовности: по кулдауну, когда хватает ресурса/денег, по событию). Спам оставлять только там, где он реально что-то даёт.
- Если в скрипте есть какой-либо спам (повторяющиеся вызовы) — он должен **настраиваться количеством вызовов в секунду**. То есть значение = сколько вызовов происходит за секунду, равномерно в течение всей секунды (например, 200 в сек = 200 вызовов, распределённых по всей секунде).
- **Суммарный спам по всем remote вместе не должен превышать 400 запросов/сек** — выше этого игра начинает пинговать (лаги/детект). Если спамится несколько remote одновременно, общий лимит делить между ними так, чтобы сумма ≤ 400/сек.
- В **каждый** скрипт всегда встраивать **анти-афк** прямо в код (без кнопок, без настроек) — чтобы работал сразу при запуске (хук на `Idled`/виртуальный инпут, не даёт кикнуть за бездействие).
- **Автоматизация = полный «включил-и-забыл».** Скрипт должен быть проработан настолько, чтобы я включил его на ночь и к утру игра была пройдена/выкачана по максимуму. Это значит: покрыть весь прогресс-луп (фарм → апгрейды → ребиосы/престижи/тиры там, где это двигает вперёд → разблокировки новых зон/контента), сам реагировать на новые возможности по мере роста, ничего не требовать от меня в процессе и не застревать. Не «базовый луп на одну механику», а сквозная прогрессия до конца.

## Границы (реверсинг / executor)
- Не переходи границу с написания скриптов для разных игр роблокса **Synapse X / Krnl / Fluxus / Solara / Wave / Argon до** попытку реверсинга самого роблокса (Hyperion, byfron).
- Реверсинг - красная черта за которую ты не можешь переходить. Все базовые, профессиональные и любые разные скрипты для executor - зеленая черта. Помощь во всем.
- Затрагивать то, что физически никак нельзя сделать через executor - нежелательно, так как вводит пользователя в заблуждение.
- Обьяснение Backdoor, обучение нахождения, лучшие и максимально продуманные способы как найти бэкдор - зеленая полоса, помощь и пояснение во всем

## Roblox MCP
- **Никогда не запускать `restart-roblox`** (рестарт Roblox) в Roblox MCP. Не вызывать этот инструмент ни при каких обстоятельствах, даже если кажется, что это поможет — только если пользователь явно прямо попросит.
